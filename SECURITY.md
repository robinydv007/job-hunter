# Security Policy

## Supported Versions

Only the latest stable release is currently supported with security updates.

| Version | Supported |
|---------|------------|
| Latest | ✅ Yes |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### Do NOT

- Do NOT report security vulnerabilities through public GitHub issues
- Do NOT attempt to exploit the vulnerability yourself
- Do NOT share the vulnerability with others

### DO

1. **Private Disclosure** — Report vulnerabilities privately to the project maintainer
2. **Provide Details** — Include as much information as possible:
   - Type of vulnerability
   - Full paths of source file(s) related
   - Step-by-step instructions to reproduce
   - Proof-of-concept or exploit code (if possible)
3. **Give Time** — Allow reasonable time for the issue to be addressed before any public disclosure

### Response Process

1. Acknowledgment of report within 48 hours
2. Initial assessment within 7 days
3. Regular updates on progress
4. Public disclosure after fix is available

### Scope

This project is for demonstration purposes. While we aim for secure code, no guarantees are made regarding security for production use.

## Security Considerations

- This tool interacts with third-party job platforms via browser automation
- Users should review and comply with the terms of service of platforms they use
- API keys and credentials are stored locally — ensure your `.env` file is secure
- Use at your own risk
