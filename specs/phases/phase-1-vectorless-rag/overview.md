# Phase 1: Vectorless RAG Chatbot
**Status**: Complete
**Depends On**: Phase 0 (Bootstrap) — complete
**Estimated Effort**: 1-2 days

## Summary
Build a simple vectorless RAG (Retrieval-Augmented Generation) application where users upload documents and ask questions. The chatbot retrieves relevant text using keyword-based search (BM25, no embeddings) and generates answers using an LLM.

## Scope
### In Scope
* Document upload (PDF, TXT, MD files)
* Text extraction and chunking
* Keyword-based retrieval (BM25 ranking — no vector database)
* Chat interface for asking questions
* LLM-powered answer generation with source citations
* Simple Streamlit web UI

### Out of Scope
* Vector embeddings or vector databases
* Multi-user authentication
* Document versioning or history
* Persistent chat history across sessions
* Deployment/infrastructure setup

## Key Deliverables
| # | Deliverable | Description |
|---|-------------|-------------|
| 1 | Document processor | Extract text from PDF/TXT/MD, chunk into segments |
| 2 | BM25 retriever | Keyword-based search over document chunks |
| 3 | LLM answer generator | Send query + context to LLM, return answer with sources |
| 4 | Streamlit UI | Upload documents, ask questions, view answers |
| 5 | Project setup | requirements.txt, .env.example, README |

## Acceptance Criteria
* [x] User can upload one or more documents (PDF, TXT, MD)
* [x] Text is extracted and chunked automatically
* [x] User can ask questions in a chat interface
* [x] Answers are generated by LLM based on document content
* [x] Answers include source references (which document/chunk)
* [x] No vector database or embeddings used (purely keyword-based)
* [x] Application runs locally with `streamlit run app.py`
