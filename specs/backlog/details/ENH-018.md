# ENH-018: Review Answers Before Apply

## Overview

Currently, when auto-apply is enabled, users confirm applying to a job WITHOUT seeing the answers that will be submitted to Naukri. This enhancement adds the ability for users to:
1. View LLM-generated answers before submission
2. Manually edit specific answers
3. Regenerate answers with additional input

## Current Flow

```
1. User sees job (title, company, score)
2. User confirms: y/n/q
3. LLM generates answers (hidden from user)
4. Submit to Naukri
```

## Proposed Flow

```
1. User sees job (title, company, score)
2. Generate LLM answers (display to user in table)
3. User choices:
   [s] Submit these answers
   [e] Edit specific answer(s)
   [r] Regenerate with input: "..."     
   [n] Skip this job
   [q] Quit
4. If Edit: Show each answer, let user modify
5. If Regenerate: Get input, call LLM again with context
6. If Submit: Send to Naukri
```

### Review Display
```
┌────┬────────────────────────────────────────────┬─────────────┐
│ #  │ Question                                   │ Answer      │
├────┼────────────────────────────────────────────┼─────────────┤
│ 1  │ TypeScript experience                      │ 5 years     │
│ 2  │ Langchain experience                       │ 2 years     │
│ 3  │ Current CTC                                │ 22 LPA      │
│ 4  │ Expected CTC                               │ 30 LPA      │
│ 5  │ Notice period                              │ Immediate   │
└────┴────────────────────────────────────────────┴─────────────┘
Submit [s] / Edit [e] / Regenerate [r] / Skip [n] / Quit [q]: 
```

### Edit Flow
```
Which answer to edit? (number): 3
Current: 22 LPA
New value: 25 LPA
Update another? [y/n]: n
```

### Regenerate Flow
```
Additional context for LLM (or Enter to skip):
>> I have 3 years not 5 in TypeScript, please update
[LLM regenerates with new context...]
[Show new answers, loop back to confirmation]
```

## Implementation Plan

### Phase 1: Config & Split Functions
- Add `review_before_apply: bool` to `AutoApplyConfig` (default: true)
- Split `naukri_apply.py`:
  - `generate_answers()` - Get LLM answers (no API calls)
  - `submit_answers()` - Submit via /respond → /apply

### Phase 2: UI Flow
- Add `display_answers_table()` - Pretty print with Rich
- Add `get_user_decision()` - Prompt for s/e/r/n/q
- Add `prompt_edit_answers()` - Let user modify specific answers

### Phase 3: Regeneration
- Add `regenerate_with_input()` - Call LLM with additional context
- Loop back to review screen after regeneration
