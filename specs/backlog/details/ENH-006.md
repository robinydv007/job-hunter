# Enhancement: Anti-Blocking — Randomize Timing

## Problem
All delays and scroll behavior in `scrape_jobs_from_page()` use fixed values:
- `asyncio.sleep(3)` after page load
- `asyncio.sleep(2)` between scrolls
- Always scrolls exactly 600px, exactly 3 times
- **Zero delay between queries** in `search_naukri()` loop — queries fire back-to-back

Bot detectors flag predictable timing patterns.

## Solution

### 1. Random delay between queries (in `search_naukri`)
```python
import random
# ... after each query completes:
await asyncio.sleep(random.uniform(3, 8))  # 3-8 second random delay
```

### 2. Jitter on all sleeps (in `scrape_jobs_from_page`)
```python
await asyncio.sleep(random.uniform(2, 5))   # was fixed sleep(3)
# ...
await asyncio.sleep(random.uniform(1.5, 3))  # was fixed sleep(2)
```

### 3. Randomize scroll behavior
```python
scroll_count = random.randint(2, 4)          # was fixed 3
for _ in range(scroll_count):
    await page.evaluate(f"window.scrollBy(0, {random.randint(400, 800)})")
    await asyncio.sleep(random.uniform(1.5, 3))
```

### 4. Config (optional, for user tuning)
```python
class SearchConfig(BaseModel):
    # ...
    delay_min_seconds: float = 3.0   # min delay between queries
    delay_max_seconds: float = 8.0   # max delay between queries
```

## Files Changed
- `src/job_hunter/search/naukri.py` — add random delays, jitter, randomized scroll
- `src/job_hunter/config.py` — optional `delay_min_seconds` / `delay_max_seconds` fields

## Testing
- Run search with 5+ queries, verify each has different delay
- Verify scroll count varies between queries
- Verify no two consecutive page loads have identical timing

## Acceptance Criteria
- No fixed sleep values — all use `random.uniform()`
- No fixed scroll count or distance — all randomized
- Minimum 3s delay between queries (configurable)
- Total scrape time increases slightly but stays reasonable (<2 min for 10 queries)
