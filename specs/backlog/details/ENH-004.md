# Enhancement: Smart Job Freshness Filter

## Problem
- `days_old=3` is hardcoded in `graph/nodes.py:86`, never configurable
- No tracking of last run/scrape activity
- User cannot control job freshness preference
- Results may include stale jobs or miss recent ones depending on run cadence

## Naukri Freshness Values (`dd` URL param)
| Value | Meaning |
|-------|---------|
| 1 | Last 1 day |
| 3 | Last 3 days |
| 7 | Last 7 days |
| 15 | Last 15 days |
| 30 | Last 30 days |

## Solution

### 1. Config — add `freshness` to `SearchConfig`
```python
freshness: int = 0  # 0=auto, 1/3/7/15/30=days
```

### 2. Run history tracking — `data/run_history.json`
Track each pipeline run:
```json
[
  { "timestamp": "2026-04-05T08:28:04", "freshness_used": 3, "jobs_found": 42, "platform": "naukri" }
]
```

### 3. Auto-calculation logic (`resolve_freshness()`)
- Always auto-calculate from last run history
- If `config.search.freshness > 0` → use `max(user_value, auto_value)`
  - Example: user sets `freshness=7`, last run was 1 day ago (auto=1) → use `dd=7`
  - Example: user sets `freshness=3`, last run was 10 days ago (auto=15) → use `dd=15`
- If `0` (auto only):
  - Load last run timestamp from `data/run_history.json`
  - If no history → default to `dd=7`
  - Calculate `days_since = (now - last_run).days`
  - Map to smallest Naukri freshness value `>= days_since`
  - Example: 2 days ago → `dd=3`, 5 days ago → `dd=7`, 20 days ago → `dd=30`

### 4. Wire into search flow
- `graph/nodes.py:search_jobs_node` — call `resolve_freshness()`, pass to `search_naukri()`
- After search completes, append run record to `data/run_history.json`

### 5. Update `config/user.yaml`
```yaml
search:
  freshness: 0  # 0=auto, 1=today, 3=3days, 7=week, 15=15days, 30=month
```

## Files Changed
- `src/job_hunter/config.py` — add `freshness` field
- `src/job_hunter/graph/nodes.py` — add `resolve_freshness()`, record run history
- `src/job_hunter/search/naukri.py` — accept `freshness` param
- `config/user.yaml` — add freshness field
- `data/run_history.json` — new file, initial `[]`

## Acceptance Criteria
- User can set explicit freshness in config
- Auto mode correctly calculates from last run history
- First run defaults to `dd=7`
- Run history is persisted and updated after each search
- Existing behavior preserved (auto mode with no history ≈ previous hardcoded `dd=3` is improved to `dd=7`)
