# ENH-016: Fix Naukri pagination using UI navigation

**Status**: resolved  
**Priority**: P1  
**Phase**: 1 (enhancement after MVP)  
**Date**: 2026-04-07  
**Completed**: 2026-04-07
The current Naukri scraper uses direct URL navigation for pagination (e.g., `/keyword-jobs-in-location-2` for page 2). However, Naukri is a Single Page Application (SPA) where direct URL navigation doesn't properly trigger the JavaScript to load new job listings. This results in:
- Page 2, 3, etc. showing the same results as page 1
- Duplicate job listings being scraped
- Wasted scraping efforts

## Root Cause
Naukri's job listings are loaded dynamically via JavaScript from their internal API. When navigating directly to pagination URLs, the SPA doesn't update the DOM with new content - it retains the initial page's job listings.

## Solution
Instead of direct URL navigation, use Naukri's built-in pagination UI controls:
1. Load page 1 using direct URL (this works correctly)
2. For subsequent pages, click the "Next" button in the pagination section
3. Wait for the page to fully load new content
4. Extract job listings
5. Repeat until desired number of pages reached

The pagination HTML structure shows:
```html
<div class="styles_pagination-cont__sWhS6">
  <div class="styles_pagination__oIvXh" id="lastCompMark">
    <!-- Previous button -->
    <a class="styles_btn-secondary__2AsIP styles_previous__PobAs">Previous</a>
    <!-- Page numbers -->
    <div class="styles_pages__v1rAK">
      <a href="/keyword-jobs-in-location">1</a>
      <a href="/keyword-jobs-in-location-2" class="styles_selected__j3uvq">2</a>
      <a href="/keyword-jobs-in-location-3">3</a>
      ...
    </div>
    <!-- Next button -->
    <a class="styles_btn-secondary__2AsIP">Next</a>
  </div>
</div>
```

## Implementation Plan

### Changes to `src/job_hunter/search/naukri.py`:

1. **Add `_click_next_button()` helper function**:
   - Finds the "Next" button by text content
   - Checks if button is disabled
   - Clicks the button with retry mechanism (3 attempts)
   - Returns True if successful, False otherwise

2. **Modify `scrape_jobs_from_page()` pagination logic**:
   - **Page 1**: Load using existing direct URL approach
   - **Pages 2-N**: 
     - Attempt to click "Next" button (with 3 retries)
     - If click fails, fallback to direct URL navigation
     - Wait for page to load new content
     - Extract jobs from updated DOM
   - **Termination conditions**:
     - No more "Next" button or button is disabled
     - Duplicate job content detected (comparing job IDs)
     - Reached max_pages limit

3. **Add duplicate detection**:
   - Track job IDs from previously scraped pages
   - Stop pagination if current page contains only duplicate jobs

### Benefits
- ✅ Eliminates duplicate job listings
- ✅ Mimics real user behavior (clicking pagination)
- ✅ Works with Naukri's SPA architecture
- ✅ Includes fallback to direct URL if clicking fails
- ✅ Prevents infinite loops with duplicate detection
- ✅ Maintains backward compatibility

### Testing
- Verify page 1 loads correctly (no regression)
- Verify clicking "Next" loads new content on page 2+
- Verify fallback to direct URL works when clicking fails
- Verify duplicate detection stops scraping when no new content
- Verify all requested pages are scraped without duplicates
