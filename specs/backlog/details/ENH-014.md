# ENH-014: Refactor CSV export with single source of truth

**Status**: resolved  
**Priority**: P2  
**Phase**: 1  
**Date**: 2026-04-06

## Problem

`src/job_hunter/export/csv_export.py` had two places to maintain column definitions:
1. `MVP_COLUMNS` list — defines column order and names
2. `row` dict inside `export_to_csv()` — manually maps each value

Adding/removing a column required updating both places, risking inconsistency.

## Solution

Replace dual definition with single `ROW_MAPPING` dict:
- Key = column name
- Value = lambda that extracts value from scored job object

```python
ROW_MAPPING: dict[str, callable[[Any], Any]] = {
    "job_title": lambda sj: _get_nested(sj.get("job", {}), "title"),
    "match_score": lambda sj: sj.get("match_score", 0),
    # ...
}
```

Export loop becomes:
```python
for sj in scored_jobs:
    row = {col: fn(sj) for col, fn in ROW_MAPPING.items()}
    writer.writerow(row)
```

## Files Changed

- Modified: `src/job_hunter/export/csv_export.py`

## Benefits

- Add/remove a column in one place only
- No `extrasaction="ignore"` needed — only writes defined columns
- Column order controlled by dict insertion order (Python 3.7+)
- Easy to add conditional logic per column via lambda
- Cleaner code: ~58 lines → ~47 lines

## Result

CSV export now has single source of truth for all columns.
