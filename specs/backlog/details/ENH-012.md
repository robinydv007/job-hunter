# Enhancement: Refactor nodes.py - Extract platform-specific code

## Problem

`src/job_hunter/graph/nodes.py` contains Naukri-specific logic mixed with generic pipeline nodes:

- `search_jobs_node()` calls `search_naukri()` directly
- Naukri-specific parameters (`days_old`, `max_jobs_per_query`) are hardcoded
- When adding new job boards (LinkedIn, Hirist, Instahyre, Foundit), this file will become bloated with platform-specific code

## Proposed Structure

```
src/job_hunter/search/
├── __init__.py                    # Export search factories
├── naukri.py                      # Naukri-specific scraper (moved from naukri.py)
├── linkedin.py                    # LinkedIn scraper (future)
├── hirist.py                      # Hirist scraper (future)
├── instahyre.py                   # Instahyre scraper (future)
├── base.py                        # Base scraper interface (new)
└── factory.py                     # Factory to get appropriate scraper (new)
```

### Base Interface (`base.py`)

```python
from abc import ABC, abstractmethod
from typing import Any
from playwright.async_api import Page
from job_hunter.resume.schema import ResumeProfile

class BaseJobScraper(ABC):
    @abstractmethod
    def search(self, profile: ResumeProfile, page: Page) -> list[dict[str, Any]]:
        """Search for jobs and return raw job listings."""
        pass
    
    @abstractmethod
    def get_platform_name(self) -> str:
        pass
```

### Factory (`factory.py`)

```python
from job_hunter.config import SearchConfig
from .base import BaseJobScraper

def get_scraper(platform: str) -> BaseJobScraper:
    scrapers = {
        "naukri": NaukriScraper,
        "linkedin": LinkedInScraper,
        # ...
    }
    return scrapers[platform]()
```

### Updated nodes.py

```python
def search_jobs_node(state: JobHunterState) -> dict:
    config = state["config"]
    profile = state["profile"]
    page = state.get("browser_page")
    
    from job_hunter.search import get_scraper
    
    all_jobs = []
    for platform in config.search.platforms:
        scraper = get_scraper(platform)
        jobs = scraper.search(profile, page)
        all_jobs.extend(jobs)
    
    return {"raw_jobs": all_jobs}
```

## Benefits

1. **Separation of concerns**: Platform logic stays with platform
2. **Scalability**: Adding new job boards = new file + register in factory
3. **Testability**: Each scraper can be unit tested independently
4. **Maintainability**: Naukri changes don't affect other platforms

## Files Changed

- `src/job_hunter/search/base.py` — new base interface
- `src/job_hunter/search/factory.py` — new factory function
- `src/job_hunter/search/naukri.py` — refactor to use base class
- `src/job_hunter/graph/nodes.py` — use factory instead of direct import

## Acceptance Criteria

- `search_jobs_node()` imports scraper via factory — no direct `from naukri import` 
- Each scraper implements `BaseJobScraper` interface
- Factory returns appropriate scraper for any registered platform
- Existing behavior unchanged — Naukri searches work identically
