# Enhancement: Expand Config for Search & Auto-Apply

## Description
`AppConfig` and `config/user.yaml` need expansion to support configurable search parameters (currently hardcoded) and auto-apply controls (needed for Phase 2 FEAT-001).

## Current Problems
- `days_old`, `max_jobs_per_query`, `max_total_jobs` are hardcoded in `nodes.py` and `naukri.py`
- No rate limiting or safety controls for auto-apply
- `ScreeningAnswers` missing ~10 common job board questions
- ENH-002 (work-mode filter) not yet implemented

## Changes

### 1. `SearchConfig` — new fields
| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `work_mode_filter` | list[str] | [] | Filter by remote/hybrid/onsite (see ENH-002) |
| `job_types` | list[str] | [] | Full-time, contract, etc. |
| `excluded_companies` | list[str] | [] | Companies to skip |
| `excluded_keywords` | list[str] | [] | e.g. "intern", "fresher", "trainee" |

Note: `days_old`, `max_jobs_per_query`, `max_total_jobs` are handled by ENH-005 (query overhaul).

### 2. New `AutoApplyConfig`
| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `enabled` | bool | False | Master toggle |
| `max_per_day` | int | 10 | Rate limit |
| `max_per_run` | int | 5 | Cap per execution |
| `delay_between_seconds` | int | 30 | Human-like delay |
| `require_confirmation` | bool | True | Prompt before apply |
| `skip_if_already_applied` | bool | True | Check past CSVs |

### 3. `ScreeningAnswers` — new fields
`current_employer`, `current_designation`, `years_in_current_role`, `highest_qualification`, `university_name`, `passing_year`, `gaps_in_employment`, `work_authorization`, `background_check_consent`, `references_available`

### 4. Wire `AppConfig`
Add `auto_apply: AutoApplyConfig` field.

### 5. Propagation
- `graph/nodes.py:search_jobs_node` — filter by exclusions post-scrape
- `config/user.yaml` — update template with all new fields + comments

## Acceptance Criteria
- All previously hardcoded values are configurable via `user.yaml`
- New fields have sensible defaults — existing behavior unchanged
- `user.yaml` template is updated with comments explaining each field
- Exclusion filters (companies, keywords) are applied post-scrape in `search_jobs_node`
