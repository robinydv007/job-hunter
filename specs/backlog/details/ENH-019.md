# ENH-019: Export Q&A to CSV

## Overview

Currently when jobs are applied via auto-apply, the screening questions and their answers are generated and submitted to Naukri, but:
1. Questions are extracted from Naukri API (`questionnaire` → `{questionId, questionName, questionType}`)
2. Answers are generated via LLM using profile + screening config
3. Neither is persisted to ScoredJob or exported to CSV

This enhancement exports Q&A data to CSV for all jobs that had screening questions.

## CSV Output Format

Single JSON column (`questionnaire`) containing array of `{question, answer}` objects:

```
questionnaire
"[{""question"":""Total years of experience"", ""answer"":""9""}, {""question"":""Current CTC"", ""answer"":""22 LPA""}]"
```

## Implementation Plan

### Phase 1: Add fields to ScoredJob (state.py)

```python
class ScoredJob(TypedDict, total=False):
    job: JobListing
    match_score: int
    matched_skills: list[str]
    why_selected: str
    apply_status: str
    apply_timestamp: str  # existing
    apply_error: str    # existing
    # NEW:
    questionnaire: str  # JSON string of [{"question": "...", "answer": "..."}]
    questionnaire_timestamp: str
```

### Phase 2: Capture Q&A during apply (nodes.py)

In `apply_node()`, after `apply_to_job()` returns:

```python
# If applied with questions, capture Q&A
if result.status == "Applied" and questions:
    qa_list = [{"question": q["name"], "answer": answers.get(q["id"], "")} 
              for q in questions]
    scored_job["questionnaire"] = json.dumps(qa_list)
    scored_job["questionnaire_timestamp"] = datetime.now().isoformat()
```

### Phase 3: Export to CSV (csv_export.py)

Add to ROW_MAPPING:

```python
"questionnaire": lambda sj: sj.get("questionnaire", "[]"),
```
