# Tech Debt: Prune Unused Fields from `JobHunterState`

## Description
`JobHunterState` in `src/job_hunter/graph/state.py` contains 11 fields, but 3 are either never read or only written without any downstream consumer:

| Field | Issue |
|-------|-------|
| `profile_validated` | Set to `True` by `load_config_node`, never read by any node |
| `messages` | LangGraph `add_messages` annotated field, never written or read by any node |
| `errors` | Only written in `search_jobs_node` error paths, never handled downstream |

## Impact
- Unnecessary state bloat makes the state harder to understand and maintain
- `messages` adds LangGraph message-reducer overhead for no benefit
- `errors` silently swallowed — if a search error occurs, the pipeline continues with empty results and the user never sees the error

## Proposed Fix
1. **Remove `profile_validated`** — no node gates on it; if validation is needed later, add it then
2. **Remove `messages`** — not a conversational agent; drop the import and annotation
3. **Remove `errors`** — replace with proper exception handling that raises and stops the graph, or use LangGraph's built-in error handling
4. **Keep `csv_path`** — useful as final output, but consider returning it from `graph.invoke()` result instead of storing in state
