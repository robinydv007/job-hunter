# Enhancement: Pan-India Fallback Query

## Problem
When user configures fewer locations than `max_locations` (e.g., 2 locations with `max_locations: 3`), the query builder only generates role×location combos. This misses jobs posted without a location filter or tagged as "All India" / "Remote".

Additionally, when `preferred_locations` is empty, the system should search Pan-India for each role — but this behavior needs to be explicit and consistent.

## Solution

### Query building logic
```python
roles = profile.target_roles[:config.max_roles]
locations = profile.preferred_locations[:config.max_locations]

queries = []

if locations:
    # Build role × location queries
    for role in roles:
        for loc in locations:
            queries.append({"keyword": role, "location": loc})
    
    # If locations < max_locations, add one Pan-India query per role
    if len(locations) < config.max_locations:
        for role in roles:
            queries.append({"keyword": role, "location": ""})
else:
    # No locations configured — Pan-India search for each role
    for role in roles:
        queries.append({"keyword": role, "location": ""})
```

### Examples

| Config | Queries generated |
|--------|-------------------|
| `preferred_locations: []`, `max_locations: 3`, 2 roles | 2 queries (role-only, Pan-India) |
| `preferred_locations: ["Bangalore"]`, `max_locations: 3`, 2 roles | 4 queries (2×Bangalore + 2×Pan-India) |
| `preferred_locations: ["Bangalore", "Pune", "Hyderabad"]`, `max_locations: 3`, 2 roles | 6 queries (2×3 locations, no Pan-India fallback) |

## Testing
- Empty locations → verify Pan-India queries generated
- 1 location with `max_locations: 3` → verify location-specific + Pan-India queries
- 3 locations with `max_locations: 3` → verify NO Pan-India fallback (full coverage)
- Verify no duplicate queries when Pan-India is added

## Acceptance Criteria
- Empty `preferred_locations` → one Pan-India query per role
- Locations count < `max_locations` → Pan-India query added per role
- Locations count >= `max_locations` → no Pan-India fallback
- No duplicate queries in the final list
