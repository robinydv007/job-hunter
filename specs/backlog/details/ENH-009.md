# Enhancement: Anti-Blocking — Exponential Backoff on Failures

## Problem
When a query fails (network error, temporary block, timeout), the next query runs immediately with no cooldown. Consecutive failures compound the risk of a full block.

Current behavior (`naukri.py:424-426`):
```python
except Exception as e:
    print(f"  [WARN] Query failed: {e}")
    continue  # immediately runs next query
```

## Solution

### 1. Track consecutive failures
```python
consecutive_failures = 0
max_consecutive_failures = 3
```

### 2. Exponential backoff between retries
```python
for qi, query in enumerate(queries):
    if consecutive_failures >= max_consecutive_failures:
        print(f"[ABORT] {max_consecutive_failures} consecutive failures, stopping")
        break
    
    try:
        jobs = await scrape_jobs_from_page(...)
        consecutive_failures = 0  # reset on success
        all_jobs.extend(jobs)
    except Exception as e:
        consecutive_failures += 1
        backoff = min(2 ** consecutive_failures, 30)  # 2s, 4s, 8s, max 30s
        jitter = random.uniform(0.5, 1.5)
        wait = backoff * jitter
        print(f"[WARN] Query failed ({consecutive_failures}/{max_consecutive_failures}), retrying in {wait:.1f}s")
        await asyncio.sleep(wait)
```

### 3. Config (optional)
```python
class SearchConfig(BaseModel):
    max_consecutive_failures: int = 3   # abort after N consecutive failures
    backoff_base_seconds: float = 2.0   # base for exponential calculation
    backoff_max_seconds: float = 30.0   # cap on wait time
```

## Acceptance Criteria
- First failure: 2s backoff (with jitter)
- Second consecutive failure: 4s backoff
- Third consecutive failure: 8s backoff
- Fourth consecutive failure: abort remaining queries
- Success resets failure counter to 0
- Partial results returned on abort
