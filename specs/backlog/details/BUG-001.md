# Bug: Interactive Config Prompts Missing

## Description
The PRD states: "On first run, the agent validates the config and prompts for any missing required fields — asking only once." 
Currently, the `cli.py` creates a default template during `job-hunter init`, but `load_config` simply loads it and fails or continues without an interactive terminal prompt (e.g., stopping and asking the user for their target salary if left blank).

## Impact
Users might run the pipeline with an incomplete configuration, leading to poor matching or silent failures, which breaks the "User In Control" mechanism.

## Proposed Fix
In `src/job_hunter/graph/nodes.py` (or `config.py`), implement an interactive prompt using `rich.prompt` or standard `input()` when `validate_profile` detects missing fields, then mutate and save the config file so it doesn't ask again on the next run.
