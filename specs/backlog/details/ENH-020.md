# ENH-020: Include Detailed Profile in LLM Answer Context

## Overview

`data/profile_detailed.yaml` is generated in Phase 2a and already loaded by the apply pipeline, but the auto-apply LLM prompt only uses the basic profile fields. This causes stack-specific experience such as `tech_experience` to be omitted from generated answers.

This enhancement supplements the existing profile context with detailed profile data so LLM answers can use richer evidence without overriding the basic profile.

## Proposed Behavior

- Keep `profile.json` as the primary profile source
- Append `profile_detailed.yaml` as supplemental context in the LLM prompt
- Include relevant fields:
  - `tech_experience`
  - `achievements`
  - `challenges_solved`
  - `interests`
  - `key_responsibilities`
- Truncate long lists to keep prompts compact

## Implementation Plan

### Phase 1: Prompt Context Builder

Add a helper in `src/job_hunter/apply/naukri_apply.py` that formats:

- basic profile summary
- detailed profile summary
- a clear instruction that detailed data is supplemental only

### Phase 2: Use in Answer Generation

Update `get_llm_answers()` to inject the combined profile context into the prompt.

### Phase 3: Verification

Add a unit test for the prompt context builder to ensure tech-specific years appear in the generated context.
