# Enhancement: Anti-Blocking — CAPTCHA Detection & Session Health

## Problem
Current block detection only checks for "Access Denied" in page text (`naukri.py:343`). Naukri often serves:
- CAPTCHA / "verify you're human" pages
- "Too many requests" warnings
- Redirect to login page
- Blank pages with 429/403 status codes

All of these go undetected — the scraper continues with empty results and wastes remaining queries.

## Solution

### 1. CAPTCHA detection
Check for common CAPTCHA indicators after each page load:
```python
captcha_signals = [
    "captcha", "verify you are human", "security check",
    "prove you're not a robot", "recaptcha", "hcaptcha"
]
page_text = await page.inner_text("body")
if any(signal in page_text.lower() for signal in captcha_signals):
    return BLOCKED_CAPTCHA
```

### 2. Session health check (before each query)
```python
async def check_session_health(page: Page) -> str:
    """Return 'healthy', 'blocked', or 'expired'"""
    # Navigate to a known-good page
    await page.goto("https://www.naukri.com/", wait_until="domcontentloaded", timeout=10000)
    page_text = await page.inner_text("body")
    
    if "login" in page_text.lower() and "welcome" not in page_text.lower():
        return "expired"  # session dropped, need re-login
    if any(captcha in page_text.lower() for captcha in CAPTCHA_SIGNALS):
        return "blocked"
    return "healthy"
```

### 3. HTTP status code checks
```python
response = await page.goto(url, ...)
if response.status in (403, 429, 503):
    return BLOCKED_HTTP  # rate limited or forbidden
```

### 4. Graceful abort
When a block is detected:
- Stop remaining queries immediately
- Log the block type and timestamp
- Return jobs collected so far (don't discard)
- Optionally notify user: "Naukri blocked after X queries, returning Y jobs collected"

### 5. New block types enum
```python
class BlockStatus(Enum):
    HEALTHY = "healthy"
    BLOCKED_CAPTCHA = "blocked_captcha"
    BLOCKED_HTTP = "blocked_http"
    SESSION_EXPIRED = "session_expired"
```

- CAPTCHA pages detected within first page load
- Session expiry detected before wasting queries
- HTTP 403/429/503 responses trigger graceful abort
